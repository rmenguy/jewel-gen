# Phase 3 — UI Design Contract

> Visual and interaction contract for Production Stack UI & Engine Integration. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (custom Tailwind components) |
| Preset | not applicable |
| Component library | none (hand-rolled `components/ui/*` + `components/Button.tsx`) |
| Icon library | Inline SVG (heroicons-style paths, no external dependency) |
| Font | Inter (400, 600) body + JetBrains Mono (400) debug/mono |

Source: `tailwind.config.js`, `src/index.css`, existing components.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, pill internal gap |
| sm | 8px | Compact element spacing, input padding |
| md | 16px | Default element spacing, card padding, panel gaps |
| lg | 24px | Section padding within panels |
| xl | 32px | Panel-to-panel gaps, major layout margins |
| 2xl | 48px | Page-level vertical rhythm between major sections |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Drag handle hit target: 44px height (touch-friendly reorder grip on stack layers)
- Image thumbnails in stack plan: 40px x 40px (consistent with existing queue item sizing)
- Base photo panel min-height: 320px (ensure locked image is always prominent per UI-01)

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 (regular) | 1.5 | Descriptions, follow-up edit input, debug text |
| Label | 11px | 600 (semibold) | 1.3 | Section headers, status badges, uppercase tracking-wider (matches existing `SectionLabel` pattern: `text-xs font-semibold uppercase tracking-wider text-gray-500`) |
| Heading | 18px | 600 (semibold) | 1.2 | Panel titles ("Stack Plan", "Generation Progress", "Follow-Up Edits") |
| Display | 24px | 600 (semibold) | 1.1 | Session name in header bar only |

Two weights only: 400 (regular) for body text and 600 (semibold) for all headings, labels, and display text.

Source: Existing `MannequinEngine.tsx` SectionLabel uses `text-xs font-semibold uppercase tracking-wider`. Button.tsx uses `font-medium` at 14px. App.tsx header mapped to semibold for weight constraint compliance.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#fafafa` (gray-50) | Page background, panel backgrounds |
| Secondary (30%) | `#ffffff` (white) | Cards, panels, modal surfaces, with `border border-gray-200` |
| Accent (10%) | `#4f46e5` (indigo-600) | See reserved list below |
| Destructive | `#dc2626` (red-600) | Remove layer, clear session, reset stack only |

Accent reserved for:
- Primary CTA buttons ("Execute Stack", "Lock Base Image", "Send to Production Stack")
- Active tab indicator in navigation
- Active PillButton state (bg-indigo-600 text-white)
- Step completion checkmark icons
- Progress bar fill during execution
- Drag handle active state when reordering

Additional semantic colors (existing codebase patterns):
- `#10b981` (emerald-500): Success states, completed step indicators, validation passed
- `#f59e0b` (amber-500): Warning states, retrying step indicator, validation marginal
- `#6b7280` (gray-500): Disabled, pending, inactive text

Source: `App.tsx` uses indigo-600 throughout. `Button.tsx` variant "primary" = `bg-indigo-600`. `PillButton.tsx` active = `bg-indigo-600 text-white`.

## Layout Contract

### Production Stack — Primary Panel Layout (UI-01 through UI-06)

The Production Stack tab uses a **3-column layout** at desktop (>=1024px) collapsing to stacked single-column at mobile:

```
+-------------------+----------------------------+-------------------+
|  LEFT (300px)     |  CENTER (flex-1)           |  RIGHT (300px)    |
|                   |                            |                   |
|  Base Photo       |  Generation Result         |  Stack Plan       |
|  Output Format    |  + Progress overlay        |  + Drag Reorder   |
|  Reference Bundle |  + Follow-up input         |  + Layer details  |
|                   |                            |  + Debug view     |
+-------------------+----------------------------+-------------------+
```

This mirrors the existing `MannequinEngine.tsx` 3-panel pattern (config | preview | refinement).

**Left Panel — Session Setup**
- **Base Photo** (UI-01): Locked base image displayed at max 280px width, 1:1 aspect container with `object-contain`. Border: 2px solid indigo-600 when locked (visual "locked" signal). Label: "BASE IMAGE" uppercase.
- **Output Format** (UI-02): Row of PillButtons for aspect ratio. Dropdown for resolution. Labels use business-friendly names.
- **Reference Bundle** (UI-04): Collapsible sections grouped by role. Each reference shows 40x40 thumbnail + role label. Count badge per group.

**Center Panel — Generation & Preview**
- Large image display area (flex-1, min-h-0 per Flexbox Viewport Rule from CLAUDE.md).
- **Generation Progress** (UI-05): Overlay bar at bottom of image showing step-by-step status. Each step is a horizontal segment: `pending` (gray-200) | `executing` (indigo-600 pulse) | `validating` (amber-500) | `completed` (emerald-500) | `failed` (red-500).
- **Follow-up Edit** (UI-06): Text input bar pinned to bottom of center panel. Placeholder: "Describe an edit..." with Send button.
- When no session exists: empty state with upload zone for base image.

**Right Panel — Stack Plan & Debug**
- **Stack Plan** (UI-03): Ordered list of jewelry layers. Each layer row: drag handle (6-dot grip icon) | 40x40 product thumbnail | layer name | target zone badge | remove button. Drag-reorder via HTML5 drag-and-drop (no external library).
- **Add Layer**: Button at bottom of list opens inline form (product image upload + category select + optional zone override).
- **Debug View** (UI-08): Collapsible section at bottom of right panel, collapsed by default. Shows: references included (count + thumbnails), references excluded (count + reasons), generationConfig JSON in mono font. Only visible when session has at least one completed step.

### Navigation Changes (UI-07)

Production Stack becomes the default tab. Tab order changes to: `PRODUCTION` | `MANNEQUIN` | `CATALOG` | `BATCH`. The PRODUCTION tab label changes to `STACK` in the nav bar.

### Mannequin Engine Changes (MANN-03, MANN-04)

Add a prominent "Send to Stack" button (indigo-600, full-width) in the center panel footer bar of MannequinEngine. Clicking it:
1. Transfers `currentImage` to `useProductionStore.createStackSession(currentImage, '1:1', '1K')`
2. Switches active engine to PRODUCTION (now STACK)

### Operator Efficiency UI (OPS-01, OPS-02, OPS-03)

**Duplicate Session (OPS-01)**: Icon button (copy icon) in session header bar. Creates a clone of the current session with a new ID, resets stepStates to pending, keeps layers/format/base. `aria-label="Duplicate session"`.

**Save/Load Presets (OPS-02)**: "Save as Preset" button in session header. Opens modal with name input. Presets stored in localStorage (same pattern as `customPresets` in production store). "Load Preset" dropdown in the session setup panel.

**Generation History (OPS-03)**: Thumbnail strip below the main image in center panel. Each completed step shows a small (64x64) thumbnail. Click to view that step's output full-size. Current step has indigo-600 ring highlight.

## Drag-and-Drop Specification (UI-03)

Stack plan reorder uses native HTML5 Drag and Drop API (no library dependency):

- `draggable="true"` on each `StackLayerRow`
- `onDragStart`: set `dataTransfer` with layer ID, apply dragging visual
- `onDragOver`: determine insertion position, show drop indicator (2px indigo-400 top border)
- `onDrop`: call `reorderStackLayers(newOrderedIds)` from production store
- `onDragEnd`: clear all visual states

Reorder commits on drop (not on drag). Visual feedback is immediate via CSS classes only.
