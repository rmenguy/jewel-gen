Create a new file `services/stackEngine.ts`. This is a pure service module (no React hooks, no Zustand imports). It operates on `ProductionStackSession` objects passed by the caller (the store or component).

**Imports:**

```typescript
import {
  ProductionStackSession, StackLayer, GenerationSnapshot, StepState,
  ImageChatSession, ImageGenerationConfig, ReferenceImage, PixelFidelityResult
} from '../types';
import {
  addJewelryToExisting, fetchImageAsBase64, createImageChatSession,

```

**Function 1: `initializeStepStates(session: ProductionStackSession): void`**

Populates `session.stepStates` from `session.layers`. For each layer, creates a `StepState` with:
- `layerId`: layer.id
- `status`: 'pending'
- `currentAttempt`: 0
- `maxAttempts`: 3
- `snapshots`: []
- `approvedSnapshotIndex`: null

**Function 2: `executeStackPlan(session: ProductionStackSession, onStepUpdate: (stepIndex: number, state: StepState) => void): Promise<void>`**

The main orchestrator. Steps:
1. Set `session.status = 'executing'`
2. Call `initializeStepStates(session)` if `stepStates` is empty
3. Set `currentImage = session.baseImage`
4. Loop through `session.layers` sequentially (NEVER parallel -- rate limiting):
   a. Get the corresponding `stepState = session.stepStates[i]`
   b. Skip if `stepState.status === 'completed'` (supports resume after partial failure)
   c. Call `executeStep(session, i, currentImage)`
   d. If step succeeds: set `currentImage = snapshot.outputImage`, set `session.currentImage = currentImage`, call `onStepUpdate(i, stepState)`
   e. If step fails: set `stepState.status = 'failed'`, `stepState.error = error.message`, call `onStepUpdate(i, stepState)`, BREAK the loop (do not continue to next step)
5. After loop: set `session.status` to `'completed'` if all steps completed, otherwise leave as `'executing'`

**Function 3: `executeStep(session: ProductionStackSession, stepIndex: number, inputImage: string): Promise<GenerationSnapshot>`** (not exported -- internal helper)

Executes a single step:
1. Get `layer = session.layers[stepIndex]` and `stepState = session.stepStates[stepIndex]`
2. Set `stepState.status = 'executing'`, increment `stepState.currentAttempt`
3. Resolve product image: if `layer.productImage` starts with `'http'`, fetch via `fetchImageAsBase64` and wrap as `data:image/jpeg;base64,${raw}`. Otherwise use as-is.
4. Call `addJewelryToExisting(inputImage, productBase64, layer.productCategory, layer.blueprint, layer.dimensions)` -- this internally handles dress -> segment -> composite -> harmonize -> pixel validation (per STACK-08). The existing pipeline already enforces physical plausibility via its prompts (STACK-12) and placement stability (STACK-13 via the "do NOT modify existing jewelry" instruction).
5. Build a `GenerationSnapshot`:
   - `stepIndex`: stepIndex
   - `layerId`: layer.id
   - `prompt`: `[addJewelryToExisting] category=${layer.productCategory} zone=${layer.targetZone} step=${stepIndex+1}/${session.layers.length}`
   - `referencesUsed`: [] (addJewelryToExisting manages refs internally)
   - `referencesExcluded`: []
   - `generationConfig`: `{ imageConfig: { imageSize: session.imageSize, aspectRatio: session.aspectRatio } }`
   - `inputImage`: inputImage
   - `outputImage`: the result from addJewelryToExisting
   - `validation`: null (validation happens inside addJewelryToExisting)
   - `timestamp`: Date.now()
   - `attemptNumber`: stepState.currentAttempt
6. Push snapshot to `stepState.snapshots`
7. Set `stepState.approvedSnapshotIndex = stepState.snapshots.length - 1`
8. Set `stepState.status = 'completed'`
9. Return snapshot

**Function 4: `retryStep(session: ProductionStackSession, stepIndex: number, onStepUpdate: (stepIndex: number, state: StepState) => void): Promise<void>`**

Per STACK-09. Retries a specific step without re-running the entire stack:
1. Get the input image for this step: if `stepIndex === 0`, use `session.baseImage`. Otherwise, get `session.stepStates[stepIndex - 1].snapshots[session.stepStates[stepIndex - 1].approvedSnapshotIndex!].outputImage`.
2. Set `session.stepStates[stepIndex].status = 'retrying'`
3. Call `onStepUpdate(stepIndex, session.stepStates[stepIndex])`
4. Call `executeStep(session, stepIndex, previousImage)`
5. Update `session.currentImage = snapshot.outputImage`
6. Call `onStepUpdate(stepIndex, session.stepStates[stepIndex])`
7. If there are steps AFTER this one that were already completed, they are now stale. Set their status to `'pending'` and clear their `approvedSnapshotIndex` to null (the user must decide whether to re-run them).

**Function 5: `initFollowUpSession(session: ProductionStackSession): void`**

Per STACK-10. Creates a chat session for post-completion follow-up edits:
1. If `!session.currentImage`, throw `Error('No completed image for follow-up')`
2. Set `session.chatSession = createImageChatSession({ aspectRatio: session.aspectRatio, imageSize: session.imageSize })`
3. Set `session.status = 'follow-up'`

**Function 6: `sendFollowUpEdit(session: ProductionStackSession, userPrompt: string): Promise<string>`**

Per STACK-10, STACK-11. Sends a conversational follow-up edit:
1. If `!session.chatSession || !session.currentImage`, throw `Error('Follow-up session not initialized')`
2. Build `userParts`:
   ```typescript
   const preserveInstruction = 'You are editing a production jewelry photo. ALL existing jewelry on the model must be PRESERVED exactly as-is unless explicitly told otherwise. Do NOT remove, shift, resize, or alter any previously placed jewelry.';
   const userParts: any[] = [
     { text: `${preserveInstruction}\n\nINSTRUCTION: ${userPrompt}` },
   ];
   ```
3. On first turn (`session.chatSession.history.length === 0`), also add the current image as inlineData:
   ```typescript
   const imageData = extractBase64(session.currentImage);
   userParts.push({ inlineData: { mimeType: 'image/png', data: imageData } });
   ```
   Import `extractBase64` from geminiService for this.
4. Call `continueImageChatSession(session.chatSession, userParts)`
5. If `result.images.length > 0`:
   - Get `newImage = result.images[0].dataUri`
   - Record a `GenerationSnapshot` in `session.followUpHistory` with `stepIndex: -1`, `layerId: 'follow-up'`, `prompt: userPrompt`, empty refs, the session's generationConfig, `inputImage: session.currentImage`, `outputImage: newImage`, `validation: null`, `timestamp: Date.now()`, `attemptNumber: session.followUpHistory.length + 1`
   - Set `session.currentImage = newImage`
   - Return newImage
6. Otherwise throw `Error('No image returned from follow-up edit')`

**Function 7: `compactSnapshots(session: ProductionStackSession): void`**

Per Pitfall 3 from research. Manages memory by clearing non-approved snapshot images:
1. For each `stepState` in `session.stepStates`:
   - For each `snapshot` in `stepState.snapshots`:
     - If the snapshot index is NOT `stepState.approvedSnapshotIndex`:
       - Set `snapshot.inputImage = ''` (cleared)
       - Set `snapshot.outputImage = ''` (cleared)
   - This keeps only the approved attempt's full images, freeing memory from failed/rejected attempts.
