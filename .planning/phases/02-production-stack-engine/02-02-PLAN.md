<objective>
Build the production stack execution engine -- the orchestrator that takes a ProductionStackSession and runs progressive sequential jewelry placement with per-step validation, retry, follow-up editing, and memory management.

Purpose: This is the core business logic of the Production Stack. It turns an ordered plan of jewelry pieces into a progressive stacking session where each step builds on the previous approved result, with full snapshot history for undo, retry, and debugging. Each step explicitly organizes references by role (STACK-06) and records them in snapshots for debugging and reproducibility.

Output: New services/stackEngine.ts file with all execution functions.

**Function 1: `buildStepBundle(session: ProductionStackSession, layer: StackLayer, inputImage: string, productBase64: string): { bundle: ReferenceBundle; prompt: string }`** (exported — STACK-06)

Constructs a ReferenceBundle with explicit roles for a single step. This documents which references play which roles, even though `addJewelryToExisting` handles the actual API call internally. The bundle is recorded in the GenerationSnapshot for debugging (UI-08) and reproducibility (STATE-04).

Implementation:
1. Create a `ReferenceImage` for the base scene:
   ```typescript
   const baseRef: ReferenceImage = {
     id: `step-${layer.ordinal}-base`,
     kind: 'character',
     role: 'locked base scene — do NOT modify existing content',
     base64: extractBase64(inputImage),
     mimeType: 'image/png',
     priority: 0,  // highest priority
   };
   ```
2. Create a `ReferenceImage` for jewelry fidelity:
   ```typescript
   const jewelryRef: ReferenceImage = {
     id: `step-${layer.ordinal}-jewelry`,
     kind: 'object',
     role: `jewelry fidelity — ${layer.name} (${layer.productCategory})`,
     base64: extractBase64(productBase64),
     mimeType: 'image/jpeg',
     priority: 1,
   };
   ```
3. If `layer.ordinal > 0` (not the first piece), create a character consistency reference from the session's base image:
   ```typescript
   const characterRef: ReferenceImage = {
     id: `step-${layer.ordinal}-character`,
     kind: 'character',
     role: 'character consistency — original mannequin identity',
     base64: extractBase64(session.baseImage),
     mimeType: 'image/png',
     priority: 2,
   };
   ```
4. Assemble the `ReferenceBundle`:
   ```typescript
   const bundle: ReferenceBundle = {
     characterReferences: layer.ordinal > 0 ? [baseRef, characterRef] : [baseRef],
     objectReferences: [jewelryRef],
     compositionReferences: [],
     styleReferences: [],
   };
   ```
5. Build the prompt string: `getZonePlacementPrompt(layer.targetZone)` + ` PLACEMENT LOCK: Do NOT remove, shift, resize, or alter ANY existing jewelry already on the model. Only ADD the new piece.` + ` PHYSICS: Correct scale relative to body, realistic drape/hang, no object fusion between jewelry pieces.`
6. Return `{ bundle, prompt }`.

**Function 2: `initializeStepStates(session: ProductionStackSession): void`**

Populates `session.stepStates` from `session.layers`. For each layer, creates a `StepState` with:
- `layerId`: layer.id
- `status`: 'pending'
- `currentAttempt`: 0
- `maxAttempts`: 3
- `snapshots`: []
- `approvedSnapshotIndex`: null

**Function 3: `executeStackPlan(session: ProductionStackSession, onStepUpdate: (stepIndex: number, state: StepState) => void): Promise<void>`**

The main orchestrator. Steps:
1. Set `session.status = 'executing'`
2. Call `initializeStepStates(session)` if `stepStates` is empty
3. Set `currentImage = session.baseImage`
4. Loop through `session.layers` sequentially (NEVER parallel -- rate limiting):
   a. Get the corresponding `stepState = session.stepStates[i]`
   b. Skip if `stepState.status === 'completed'` (supports resume after partial failure)
   c. Call `executeStep(session, i, currentImage)`
   d. If step succeeds: set `currentImage = snapshot.outputImage`, set `session.currentImage = currentImage`, call `onStepUpdate(i, stepState)`
   e. If step fails: set `stepState.status = 'failed'`, `stepState.error = error.message`, call `onStepUpdate(i, stepState)`, BREAK the loop (do not continue to next step)
5. After loop: set `session.status` to `'completed'` if all steps completed, otherwise leave as `'executing'`

**Function 4: `executeStep(session: ProductionStackSession, stepIndex: number, inputImage: string): Promise<GenerationSnapshot>`** (not exported -- internal helper)

Executes a single step:
1. Get `layer = session.layers[stepIndex]` and `stepState = session.stepStates[stepIndex]`
2. Set `stepState.status = 'executing'`, increment `stepState.currentAttempt`
3. Resolve product image: if `layer.productImage` starts with `'http'`, fetch via `fetchImageAsBase64` and wrap as `data:image/jpeg;base64,${raw}`. Otherwise use as-is.
4. Call `buildStepBundle(session, layer, inputImage, productBase64)` to get the reference bundle and prompt for this step (STACK-06).
5. Call `addJewelryToExisting(inputImage, productBase64, layer.productCategory, layer.blueprint, layer.dimensions)` -- this internally handles dress -> segment -> composite -> harmonize -> pixel validation (per STACK-08). The existing pipeline already enforces physical plausibility via its prompts (STACK-12). For STACK-13 (placement locking), `dressWithJewelry` says "ONLY add the jewelry" and the pixel validation corrections say "Do NOT modify existing jewelry." The 2-pass canvas compositing also preserves existing pixels by design.
6. Build a `GenerationSnapshot`:
   - `stepIndex`: stepIndex
   - `layerId`: layer.id
   - `prompt`: the prompt string from `buildStepBundle` (records the full placement + lock + physics prompt for STATE-04)
   - `referencesUsed`: `[...stepBundle.bundle.characterReferences, ...stepBundle.bundle.objectReferences]` (STACK-06 — explicit role documentation)
   - `referencesExcluded`: [] (no budget enforcement needed at this level — addJewelryToExisting takes exactly 2 images)
   - `generationConfig`: `{ imageConfig: { imageSize: session.imageSize, aspectRatio: session.aspectRatio } }`
   - `inputImage`: inputImage
   - `outputImage`: the result from addJewelryToExisting
   - `validation`: null (validation happens inside addJewelryToExisting; result not surfaced yet)
   - `timestamp`: Date.now()
   - `attemptNumber`: stepState.currentAttempt
7. Push snapshot to `stepState.snapshots`
8. Set `stepState.approvedSnapshotIndex = stepState.snapshots.length - 1`
9. Set `stepState.status = 'completed'`
10. Update session-level STATE-01 fields: push any validation result to `session.validationResults` if available. Set `session.referenceBundle` to the latest step's bundle (accumulates the most complete picture). Flatten all step bundles' excluded refs into `session.excludedReferences`.
11. Return snapshot

**Function 5: `retryStep(session: ProductionStackSession, stepIndex: number, onStepUpdate: (stepIndex: number, state: StepState) => void): Promise<void>`**

Per STACK-09. Retries a specific step without re-running the entire stack:
1. Get the input image for this step: if `stepIndex === 0`, use `session.baseImage`. Otherwise, get `session.stepStates[stepIndex - 1].snapshots[session.stepStates[stepIndex - 1].approvedSnapshotIndex!].outputImage`.
2. Set `session.stepStates[stepIndex].status = 'retrying'`
3. Call `onStepUpdate(stepIndex, session.stepStates[stepIndex])`
4. Call `executeStep(session, stepIndex, previousImage)`
5. Update `session.currentImage = snapshot.outputImage`
6. Call `onStepUpdate(stepIndex, session.stepStates[stepIndex])`
7. If there are steps AFTER this one that were already completed, they are now stale. Set their status to `'pending'` and clear their `approvedSnapshotIndex` to null (the user must decide whether to re-run them).

**Function 6: `initFollowUpSession(session: ProductionStackSession): void`**

Per STACK-10. Creates a chat session for post-completion follow-up edits:
1. If `!session.currentImage`, throw `Error('No completed image for follow-up')`
2. Set `session.chatSession = createImageChatSession({ aspectRatio: session.aspectRatio, imageSize: session.imageSize })`
3. Set `session.status = 'follow-up'`

**Function 7: `sendFollowUpEdit(session: ProductionStackSession, userPrompt: string): Promise<string>`**

Per STACK-10, STACK-11. Sends a conversational follow-up edit:
1. If `!session.chatSession || !session.currentImage`, throw `Error('Follow-up session not initialized')`
2. Build `userParts`:
   ```typescript
   const preserveInstruction = 'You are editing a production jewelry photo. ALL existing jewelry on the model must be PRESERVED exactly as-is unless explicitly told otherwise. Do NOT remove, shift, resize, or alter any previously placed jewelry.';
   const userParts: any[] = [
     { text: `${preserveInstruction}\n\nINSTRUCTION: ${userPrompt}` },
   ];
   ```
3. On first turn (`session.chatSession.history.length === 0`), also add the current image as inlineData:
   ```typescript
   const imageData = extractBase64(session.currentImage);
   userParts.push({ inlineData: { mimeType: 'image/png', data: imageData } });
   ```
4. Call `continueImageChatSession(session.chatSession, userParts)`
5. If `result.images.length > 0`:
   - Get `newImage = result.images[0].dataUri`
   - Record a `GenerationSnapshot` in `session.followUpHistory` with `stepIndex: -1`, `layerId: 'follow-up'`, `prompt: userPrompt`, empty refs, the session's generationConfig, `inputImage: session.currentImage`, `outputImage: newImage`, `validation: null`, `timestamp: Date.now()`, `attemptNumber: session.followUpHistory.length + 1`
   - Set `session.currentImage = newImage`
   - Return newImage
6. Otherwise throw `Error('No image returned from follow-up edit')`

**Function 8: `compactSnapshots(session: ProductionStackSession): void`**

Per Pitfall 3 from research. Manages memory by clearing non-approved snapshot images:
1. For each `stepState` in `session.stepStates`:
   - For each `snapshot` in `stepState.snapshots`:
     - If the snapshot index is NOT `stepState.approvedSnapshotIndex`:
       - Set `snapshot.inputImage = ''` (cleared)
       - Set `snapshot.outputImage = ''` (cleared)
   - This keeps only the approved attempt's full images, freeing memory from failed/rejected attempts.
